# คำสั่งสำหรับวางใน Claude Code

## ขั้นตอน
1. ดาวน์โหลด `fluid-app.zip` แล้วแตกไฟล์ไว้ในเครื่อง
2. เปิด terminal เข้าไปในโฟลเดอร์นั้น: `cd fluid-app`
3. เปิด Claude Code: `claude`
4. ก๊อปคำสั่งข้างล่างนี้ไปวาง

---

## คำสั่งที่ 1 — เอาแอปขึ้นเว็บให้ใช้บนมือถือ (ทำก่อน)

นี่คือ PWA บันทึกการดื่มน้ำและปัสสาวะรายวัน เป็น static site ล้วน ไม่ต้อง build

ช่วยทำให้หน่อย:
1. รัน local server แล้วเช็คว่า index.html, manifest.webmanifest, sw.js และไอคอนโหลดครบไม่มี error
2. ตรวจว่า PWA ติดตั้งได้จริง (manifest ถูกต้อง, service worker register ผ่าน, มีไอคอน 192 กับ 512)
3. deploy ขึ้น GitHub Pages ให้ โดยสร้าง repo ใหม่ชื่อ fluid-balance แบบ public
4. บอก URL ที่ได้ พร้อมขั้นตอนกดเพิ่มลงหน้าจอโฮมบน iPhone และ Android

ถ้ามีอะไรที่ทำให้ติดตั้งลงหน้าจอโฮมไม่ผ่าน แก้ให้เลย

---

## คำสั่งที่ 2 — เพิ่มฟีเจอร์ (ทำทีละอัน อย่าสั่งรวดเดียว)

อ่าน index.html ให้เข้าใจโครงสร้างก่อน แล้วเพิ่มฟีเจอร์นี้โดยรักษาสไตล์เดิมไว้
(กระบอกตวง 2 อัน สีฟ้า #0E7FBF สำหรับน้ำดื่ม สีเหลือง #CE9412 สำหรับปัสสาวะ ฟอนต์ IBM Plex Sans Thai + Mono):

- ตั้งเป้าหมายน้ำต่อวัน แล้วโชว์เส้นเป้าหมายบนกระบอกตวงฝั่งน้ำดื่ม
- กราฟย้อนหลัง 7 วัน เทียบน้ำเข้ากับน้ำออก
- ส่งออกเป็นไฟล์ CSV เอาไปเปิดใน Excel ให้หมอดูได้
- ปุ่มสำรองข้อมูล/กู้คืนเป็นไฟล์ JSON กันข้อมูลหายตอนล้างเบราว์เซอร์
- บันทึกสีปัสสาวะ (ใส / เหลืองอ่อน / เหลืองเข้ม / น้ำตาล) เป็นตัวเลือกกดง่าย ๆ ตอนกรอก

ทำทีละข้อ ทำเสร็จข้อหนึ่งให้หยุดถามก่อนว่าโอเคไหม แล้วค่อยทำข้อถัดไป
ทุกครั้งที่แก้เสร็จ ให้เปิดดูใน browser ขนาดจอมือถือ (390x844) แล้วยืนยันว่าปุ่มยังกดง่ายและไม่ล้นจอ

---

## คำสั่งที่ 3 — ถ้าอยากให้ข้อมูลซิงก์ข้ามเครื่อง

ตอนนี้ข้อมูลเก็บใน localStorage อยู่ในเครื่องเดียว
ช่วยประเมินทางเลือกให้หน่อยว่าถ้าอยากให้เปิดจากมือถือกับคอมแล้วเห็นข้อมูลเดียวกัน
ควรใช้อะไร (เช่น Cloudflare D1, Supabase, หรือแค่ sync ผ่านไฟล์)
เทียบข้อดีข้อเสีย ความยาก และค่าใช้จ่าย แล้วเสนอทางที่ง่ายที่สุดสำหรับ static site ตัวนี้
อย่าเพิ่งเขียนโค้ด รอผมเลือกก่อน
